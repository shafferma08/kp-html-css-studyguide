Question:
Which DOCTYPE declaration is used to specify an HTML5 document?

Options:

  • <!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN">
  • <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN">
  • <!DOCTYPE html>
  • <!DOCTYPE HTML5>
  • <?DOCTYPE version="5.0"?>

Correct Answer:

  • <!DOCTYPE html>

Explanation:
The <!DOCTYPE html> declaration is used to define the document type and version of HTML. For HTML5 documents, this is the correct declaration and it tells the browser to render the page in standards mode.

Example:

<!DOCTYPE html>
<html>
<head>
  <title>My HTML5 Document</title>
</head>
<body>
  <p>Hello, World!</p>
</body>
</html>

In this HTML5 example, the <!DOCTYPE html> declaration at the top of the file is the standard way to indicate to the browser that the following code should be interpreted as an HTML5 document.

Question:
In an HTML table, which tag denotes a single row?

Options:

  • <rw>
  • <row>
  • <td>
  • <line>
  • <tr>

Correct Answer:

  • <tr>

Explanation:
In HTML, the <tr> tag is used to define a table row. A table row can contain one or more <td> (table data/cell) or <th> (table header) elements.

Example:

<table border="1">
  <tr>
    <td>Row 1, Cell 1</td>
    <td>Row 1, Cell 2</td>
  </tr>
  <tr>
    <td>Row 2, Cell 1</td>
    <td>Row 2, Cell 2</td>
  </tr>
</table>

In this example, each <tr> element creates a new row in the table, and the <td> elements within define the individual cells of that row.

Question:
Which of the following shows a correctly formatted tag to display a hyperlink?

Options:

  • <a href="www.google.com"> Google </a>
  • <a href=www.google.com>Google</a href>
  • <a="www.google.com">Google</a>
  • <a ref="www.google.com">Google</a>

Correct Answer:

  • <a href="www.google.com"> Google </a>

Explanation:
The anchor <a> tag is used to define a hyperlink, which is used to link from one page to another. The correct format of the anchor tag includes the href attribute, which specifies the URL of the page the link goes to.

Example:

<!-- Correctly formatted hyperlink -->
<a href="https://www.google.com">Visit Google</a>

In this example, the <a> tag has an href attribute with a value that is the URL to which the link points. The text “Visit Google” is what will be clickable by the user.

Question:
Which of the following is NOT a valid value of the display property in CSS?

Options:

  • inline-grid
  • block-inline
  • flex
  • inline-table
  • table-row

Correct Answer:

  • block-inline

Explanation:
The value block-inline is not a valid value for the display property in CSS. The correct value that combines block and inline behaviors is inline-block. The flex value is indeed valid and is used for the CSS Flexible Box Layout, commonly known as Flexbox.

Example:

/* Correct use of the display property */
.element {
  display: inline-block; /* Correct value */
  display: flex;         /* Correct value */
  display: inline-table; /* Correct value */
  display: table-row;    /* Correct value */
}

/* Incorrect use of the display property */
.incorrect-element {
  display: block-inline; /* This is not valid */
}

In this example, .element class has multiple valid display properties as comments, while .incorrect-element class tries to use an invalid value.

Question:
Which HTML tag is used to make text italic?

Options:

  • <i>
  • <italic>
  • <em>
  • <italics>
  • <ital>

Correct Answer:

  • <i>

Explanation:
The <i> tag is used to make text italic in HTML. This tag is often used to indicate a range of different text qualities including technical terms, phrases from other languages, or thought. The <em> tag also italicizes text but is semantically used to give emphasis to the text, which could be reflected by screen readers and other assistive technologies.

Example:

<p>This is normal text - <i>this is italic text</i>.</p>
<p>This is normal text - <em>this is emphasized text that is also italicized</em>.</p>

In this example, text within <i> tags is displayed in italics. Similarly, text within <em> tags is also italicized but semantically indicates that the text has an added emphasis.

Question:
If a DOCTYPE declaration is omitted from an HTML document, how might some browsers interpret the page?

Options:

  • In strict mode, adhering to the latest HTML standards.
  • In quirks mode, using fallback rendering behavior.
  • In HTML5 mode by default.
  • As an XML document rather than an HTML document.
  • With JavaScript functionality disabled.

Correct Answer:

  • In quirks mode, using fallback rendering behavior.

Explanation:
Omitting the DOCTYPE declaration can cause browsers to render the page in “quirks mode.” In this mode, browsers use a different set of rules for rendering, which often mimics the behavior of older browsers for compatibility with web pages that do not include a DOCTYPE. This can result in inconsistent styling and layout across different browsers.

Example:
A web page without a DOCTYPE might render differently in various browsers because each browser might have its own quirks mode behavior:

<!-- No DOCTYPE declaration, which could trigger quirks mode -->
<html>
<head>
  <title>My Web Page</title>
</head>
<body>
  <p>This paragraph might be rendered inconsistently across browsers.</p>
</body>
</html>

In this example, the absence of a DOCTYPE declaration could lead to unpredictable styles and layout issues when the page is viewed in different browsers.

Question:
To create a repeating back and forth animation effect, which animation-direction value would you use?

Options:

  • alternate
  • reverse
  • backwards
  • loop
  • replay

Correct Answer:

  • alternate

Explanation:
The animation-direction property defines whether an animation should be played forwards, backwards or in alternate cycles. The alternate value makes the animation change direction after each cycle, creating a back and forth effect.

Example:

@keyframes example {
  from {background-color: red;}
  to {background-color: yellow;}
}

/* Apply the animation to a div */
div {
  width: 100px;
  height: 100px;
  background-color: red;
  animation-name: example;
  animation-duration: 4s;
  animation-iteration-count: infinite;
  animation-direction: alternate; /* Will cause the div to alternate between red and yellow */
}

<div></div>

In this CSS example, the div will alternate its background color between red and yellow every 4 seconds, indefinitely, because animation-iteration-count is set to infinite and animation-direction is set to alternate.

Question:
Select the correct code below to make the largest heading invisible on the page.

Options:

  • h1 { invisibility: false; }
  • h1 { visibility: invisible; }
  • h1 { visibility: hidden; }
  • h2 { visibility: hidden; }

Correct Answer:

  • h1 { visibility: hidden; }

Explanation:
The visibility CSS property can be used to show or hide elements without affecting the layout of the document. The hidden value hides the element, but it still takes up space in the layout. The correct syntax for this property is visibility: hidden;.

Example:

h1 {
  visibility: hidden; /* This will hide the h1 element */
}

<h1>This heading will not be visible, but it will still take up space.</h1>

In this example, the <h1> element’s content will not be visible, but the space it occupies will remain part of the page’s layout. The other options are either incorrect (invisibility: false;, visibility: invisible;) or apply to a different element (h2).

Question:
If you wanted to set the font size to adjust dynamically based on the parent element’s size, which unit would you most likely use?

Options:

  • px
  • em
  • pt
  • cm
  • inch

Correct Answer:

  • em

Explanation:
The em unit is a scalable unit that is used in web document media. It is based on the font size of the parent element. Therefore, using em allows for a scalable and responsive design that adapts to the font size of the parent element.

Example:

.parent {
  font-size: 20px; /* This sets the base font size for child elements */
}

.child {
  font-size: 1.5em; /* This will be 1.5 times the parent element's font size, so 30px */
}

<div class="parent">
  Parent text
  <div class="child">
    Child text
  </div>
</div>

In this example, the text inside the .child class will have a font size that is 1.5 times the size of the text in the .parent class due to the use of the em unit.

Question:
Hyperlink states can change depending on if they are clicked or not. Select the correct state for an unvisited link.

Options:

  • active
  • link
  • a href
  • present

Correct Answer:

  • link

Explanation:
In CSS, hyperlink states are styled using pseudo-classes. The :link pseudo-class applies to links that have not yet been visited. The :visited pseudo-class applies to links that the user has already visited. The :active pseudo-class applies to the link that is currently being clicked, and :hover applies when the user mouses over the link.

Example:

/* Unvisited link */
a:link {
  color: blue;
}

/* Visited link */
a:visited {
  color: purple;
}

/* Link being clicked */
a:active {
  color: red;
}

/* Mouse hovers over the link */
a:hover {
  color: green;
}

<a href="https://www.example.com">Example Link</a>

In this example, an unvisited link will be blue, a visited link will turn purple, the color will change to green when hovered over, and to red when actively being clicked.
